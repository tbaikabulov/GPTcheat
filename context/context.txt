Аналитик метрик поиска (Wildberries, последние 2 года)
Поддерживал дашборды в Redash и WB BI для метрик поиска (NDCG ~80%), основанных на разметке асессоров. 
Разработал ETL-процессы в Airflow, создал витрины в Clickhouse (Python, Jupyter Notebook).
Внедрил контроль качества асессоров: метрика ошибок позволила выявлять и отстранять низкокачественных работников, переоценивать их задачи.
Исследовал эффективность поисковых фильтров (напр., “футболка красная” vs “футболка” + фильтр “красный”): выявил плохо работающие фильтры, передал на доработку.
Провел исследование перехода на разметку LLM (GPT, Yandex GPT, GigaChat, Deepseek): точность 74.5–77.1%, экономия 40% расходов на разметку.
Суть проекта
Оценивал релевантность выдачи поиска Wildberries. Каждые 3 месяца собирал корзину из 3000 запросов, обкачивал топ-12 товаров (36,000 пар запрос-товар/день). Асессоры оценивали релевантность (0/1). Метрика NDCG отслеживалась ежедневно, выявлялись аномалии. Выгрузка без рекламы показывала +2% к NDCG.
Дашборд метрик поиска  
Общий NDCG (~80%).  
NDCG на популярных запросах (топ-100,000, ~96%).  
NDCG на топ-100 позиций (500 запросов).  
NDCG по текстовой релевантности (общая и пресеты).  
Доля пустых выдач (<12, <3, 0 товаров).  
Количество рекламы (~6/12 карточек).  
NDCG по категориям (Одежда, Обувь, Красота, Смартфоны).
Основной процесс  
Сбор корзины (3000 запросов/квартал, проверка по категориям/частотности).  
Обкачка выдачи (ежедневно, топ-12 товаров).  
Оценка асессорами через Markup, выгрузка в PostgreSQL.  
ETL (Python, Airflow): данные в Clickhouse, обновление дашборда.  
Анализ аномалий, правки дашборда (новые графики, корзины).
Проблемы и решения  
Сбои обкачки: внедрил алерты в Telegram.  
Ошибки асессоров: оценка с перекрытием (3 человека, 95% совпадений), метрика разногласий (5%).  
Сбои выгрузки: координация с разработчиками.
Пример задачи: Эффективность фильтров
Сравнил выдачу по запросам “футболка красная” и “футболка” + фильтр “красный”. Собрал запросы, обкачал выдачу, асессоры оценили релевантность. Посчитал NDCG, выявил неэффективные фильтры, передал команде.
Проект перехода на LLM
Оценил модели (GPT, Yandex GPT, GigaChat, Deepseek) для разметки. Точность: 74.5–77.1%. LLM может заменить 40% разметки, экономия 3 млн руб./мес.
